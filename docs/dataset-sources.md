# Dataset Sources & Licenses

## Active Datasets

### 1. NASA C-MAPSS (Turbofan Engine)
- **Name**: Commercial Modular Aero-Propulsion System Simulation
- **Source**: NASA Prognostics Center of Excellence
- **URL**: https://data.nasa.gov/dataset/C-MAPSS-Aircraft-Engine-Simulator-Data/
- **Subsets**: FD001 (single condition, single fault), FD002, FD003, FD004 (multi-condition, multi-fault)
- **License**: Public domain (US Government work)
- **Nature**: **Synthetic** — generated by NASA's C-MAPSS engine simulator
- **Used for**: Engine RUL prediction (BiLSTM + Attention model)
- **Status**: Download pending (mirror URL resolution)

### 2. UCI Hydraulic System Condition Monitoring
- **Name**: Condition monitoring of hydraulic systems
- **Source**: UCI Machine Learning Repository
- **URL**: https://archive.ics.uci.edu/dataset/447/condition+monitoring+of+hydraulic+systems
- **License**: CC BY 4.0
- **Nature**: **Real sensor data** from a hydraulic test rig
- **Sensors**: 17 physical quantities (pressure, temperature, volume flow, vibration, etc.)
- **Conditions**: 2205 measurement cycles, 4 fault types with severity levels
- **Used for**: 1D Conv Autoencoder anomaly detection
- **Status**: ✅ Downloaded and validated

### 3. Landing Gear / Brakes (Physics-Informed Synthetic)
- **Name**: AeroSentinal Landing Gear Synthetic Generator
- **Source**: Custom physics-informed generator (`scripts/generate_synthetic.py`)
- **License**: Project-internal
- **Nature**: **Synthetic** — kinetic energy model: `E = 0.5 × m × v²`, brake wear from energy absorption over landing cycles
- **Parameters**: Mass (60,000-80,000 kg), approach speed (60-75 m/s), deceleration profile, runway condition
- **Disclosure**: No public landing gear wear telemetry dataset identified. This synthetic data is clearly labeled in all API responses.
- **Used for**: XGBoost brake wear classifier

### 4. APU (Physics-Informed Synthetic)
- **Name**: AeroSentinal APU EGT Synthetic Generator
- **Source**: Custom physics-informed generator (`scripts/generate_synthetic.py`)
- **License**: Project-internal
- **Nature**: **Synthetic** — degraded EGT startup curve model with controllable fouling parameters
- **Parameters**: Fleet baseline EGT (620°C), startup ramp rate, fouling factor (0-1), ambient temperature
- **Disclosure**: No public degraded-APU telemetry dataset identified. Same approach as NASA's C-MAPSS (physics-based simulation where real degraded data is unavailable).
- **Used for**: Random Forest health scoring

### 5. ECS (Reverse-Brayton Thermodynamic Model)
- **Name**: AeroSentinal ECS Simulator
- **Source**: Custom physics model (`backend/app/services/inference.py`)
- **License**: Project-internal
- **Nature**: **Physics model** — not a dataset. Parameterized reverse-Brayton cycle with heat exchanger effectiveness degradation.
- **Parameters**: T_ambient (220K cruise), gamma (1.4), pressure ratio (3.5), nominal COP (0.85)
- **Disclosure**: This is intentionally a physics model, not ML-based. The ECS module's purpose is to demonstrate cross-domain coupling attribution.

## NLP Knowledge Base
- **Name**: AeroSentinal Curated Maintenance Pairs
- **Source**: Hand-curated from general aviation maintenance manuals and standard practices
- **Size**: 18 problem/action pairs across 5 subsystems
- **License**: Project-internal
- **Disclosure**: This is a small curated set for Phase 1 TF-IDF retrieval. Phase 2 targets MaintNet and fine-tuned LLM.
